# 実装計画

- [x] 1. 基盤: Android プロジェクト雛形と永続化スキーマ

- [x] 1.1 Android プロジェクトの雛形を作成する
  - Kotlin 2.x + Compose(Material 3) + Navigation Compose + Room(KSP) の依存関係を設定し、minSdk 33 の単一モジュール構成にする。
  - Manifest に `ACCESS_WIFI_STATE` と `ACCESS_FINE_LOCATION` を宣言し、`INTERNET` やストレージ権限は宣言しない。位置情報権限の宣言は要件 2.1 のランタイム権限要求が成立するための前提として扱う。
  - 単一 Activity、空のナビゲーション骨格、最小テーマ、Application と空の DI コンテナを用意する。
  - 完了条件: debug ビルドが成功し、空のホーム画面が実機またはエミュレータで起動できる。
  - _Requirements: 2.1_

- [x] 1.2 部屋と測定レコードの永続化スキーマを実装する
  - 部屋名の一意制約を持つ `rooms` テーブルと、12 項目を持つ `measurements` テーブルを定義する。
  - `measurements.room_id` から `rooms.id` への外部キーと `ON DELETE CASCADE` を設定する。
  - 部屋一覧、履歴の新しい順取得、部屋絞り込み、CSV エクスポート用の部屋名 JOIN 済み全件取得、測定レコード挿入の DAO クエリを定義する。
  - 同一部屋・同一日の複数測定を許可し、一意制約を設定しない。
  - インメモリ DB テストで、部屋削除時に測定レコードが連鎖削除されること、履歴が新しい順に返ること、部屋絞り込みが効くこと、エクスポート行に部屋名が含まれることを検証する。
  - 完了条件: DB 関連の instrumented test がすべて成功する。
  - _Requirements: 4.3, 4.4, 1.6, 5.1, 5.3_

- [ ] 2. コア: データ層とプラットフォーム層

- [x] 2.1 (P) 部屋管理リポジトリを実装する
  - 部屋一覧の購読、追加、名前変更、削除の操作を提供する。
  - 追加・名前変更時は trim 後に空文字と重複名を検証し、型付きエラーを返す。
  - 名前変更後も既存測定データとの関連が維持されることをテストする。
  - 削除操作が DB の CASCADE に正しく委譲されることを確認する。CASCADE 自体の検証は 1.2 が所有する。
  - 完了条件: 部屋名検証と CRUD のユニットテストが成功する。
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.6_
  - _Boundary: RoomRepository_

- [x] 2.2 (P) 測定レコード保存・購読リポジトリを実装する
  - 測定レコード保存、履歴購読、新しい順の取得、部屋絞り込み、CSV エクスポート用全件取得を提供する。
  - 履歴とエクスポート用データは部屋名 JOIN 済みで返し、測定日時順で安定して並ぶようにする。
  - 完了条件: インメモリ DB を使った保存、購読、絞り込みのテストが成功する。
  - _Requirements: 3.8, 4.1, 4.4, 5.1, 5.3_
  - _Boundary: MeasurementRepository_

- [x] 2.3 (P) Wi-Fi 接続状態の監視機能を実装する
  - 位置情報フラグ付きのネットワークコールバックを登録し、Wi-Fi 接続状態と最新の接続情報スナップショットを購読可能な形で公開する。
  - SSID のクォート除去、取得不可時の null 化、周波数(MHz)から帯域文字列(2.4GHz/5GHz/6GHz)への変換をこの層で完結させる。
  - 帯域変換の境界値ユニットテストを追加する。
  - 完了条件: 実機またはエミュレータで Wi-Fi ON/OFF に応じた状態変化をログで確認できる。
  - _Requirements: 2.4, 3.3, 3.7_
  - _Boundary: WifiStatusMonitor_

- [ ] 3. コア: サービス層

- [x] 3.1 (P) 測定セッションエンジンを実装する
  - 30 秒間、3 秒間隔で最新の Wi-Fi スナップショットを採取する。
  - 平均 RSSI は小数 1 桁、最小 RSSI、最大 RSSI、サンプル数を集計する。
  - SSID、BSSID、帯域、リンク速度は最後の有効サンプルの値を採用し、メモとともに 1 レコードとして保存する。
  - 採取中に Wi-Fi 切断を検出した場合は即座に失敗終了し、保存しない。
  - コルーチンキャンセル時も保存しない。有効サンプルが 0 件の場合も失敗とする。
  - 進捗として経過時間と採取数を公開する。
  - SSID は検証せず、そのまま記録する。
  - 完了条件: 集計、Wi-Fi 切断中断、キャンセル破棄、0 サンプル失敗のユニットテストが成功する。
  - _Requirements: 3.2, 3.3, 3.6, 3.7, 3.8, 4.1, 4.2_
  - _Depends: 2.2, 2.3_
  - _Boundary: MeasurementEngine_

- [x] 3.2 (P) 測定前提条件の判定機能を実装する
  - 位置情報権限、位置情報サービス、Wi-Fi 接続の順に固定順序で判定し、最初の不備種別を返す。
  - 権限未許可、位置情報サービス OFF、Wi-Fi 未接続、正常系の判定をユニットテストする。
  - 完了条件: 3 種類の不備と正常系を含む判定順序テストが成功する。
  - _Requirements: 2.1, 2.2, 2.3, 2.4_
  - _Depends: 2.3_
  - _Boundary: PreconditionChecker_

- [x] 3.3 (P) CSV エクスポート機能を実装する
  - 全レコードをヘッダー + 12 列で CSV 化する。列は測定日、測定時刻、部屋名、SSID、BSSID、周波数帯、平均 RSSI、最小 RSSI、最大 RSSI、サンプル数、リンク速度、メモとする。
  - 測定日時順で出力する。
  - RFC 4180 準拠のエスケープを実装し、カンマ、引用符、改行を含む値を正しく出力する。
  - UTF-8 BOM を先頭に付与する。
  - MediaStore 経由で Downloads に `wifi_log_yyyy-MM-dd.csv` として保存する。
  - レコード 0 件の場合はファイルを作成せず、型付きエラーを返す。
  - 完了条件: エスケープ、BOM、列構成、0 件エラーのユニットテストが成功する。
  - _Requirements: 6.1, 6.2, 6.3, 6.4, 6.6_
  - _Depends: 2.2_
  - _Boundary: CsvExporter_
